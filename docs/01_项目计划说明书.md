# SmartPlant EMS 项目计划说明书

## 1. 项目概述

### 1.1 项目名称
智慧工厂能源管理系统（SmartPlant EMS）

### 1.2 项目背景
随着制造业数字化转型推进，中小型生产制造企业需要精细化管控能源消耗。传统粗放的能耗管理方式导致成本高、效率低、难以追溯。本系统旨在为拥有 50-500 台设备的中小型工厂提供一套完整的能源监控、分析和优化解决方案。

### 1.3 项目目标
- 实现设备能耗实时采集和监控
- 按产线/班组/工单统计能源成本
- 提供峰谷平电价优化建议
- 设备异常预警，降低故障停机时间
- 生成多维度能耗报表，支撑决策分析

### 1.4 项目范围

**纳入范围：**
- 设备管理（CRUD、状态监控）
- 能耗数据采集与统计分析
- 告警管理
- 报表生成与导出
- 用户权限管理

**不纳入范围：**
- 设备硬件改造和 PLC 接入（仅提供 HTTP 数据接入规范）
- ERP/MES 系统集成
- 移动端 APP 开发
- 碳排放核算

### 1.5 约束条件

| 约束类型 | 具体说明 |
|---------|---------|
| **技术栈** | 后端 Beego 2.x（Go），前端 React 18 + Vite |
| **预算** | 零额外软件采购费用，使用开源组件 |
| **硬件** | 4核8G 虚拟机 × 1台 |
| **进度** | 2个月内完成 MVP（最小可行产品） |
| **团队** | 1-2 人开发 |

## 4. 风险评估

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| Go 工具链编译问题 | 高 | 中 | 使用 Docker golang:1.21 镜像编译 |
| 数据接入规范未定义清楚 | 中 | 高 | 提前与设备供应商对接 HTTP 接入规范 |
| 性能不足以支撑高频写入 | 低 | 高 | 先用 MySQL，评估后上 TimescaleDB |
| 网络不稳定导致 Git 操作失败 | 中 | 低 | 重要节点代码本地备份 |

---

## 5. 质量保障

- **代码规范**：Go 使用 `gofmt`，TypeScript 使用 ESLint
- **Git 管理**：所有代码通过 Git 管理，功能分支开发
- **自动化测试**：每个 API 接口需要有单元测试
- **文档维护**：代码内注释 + README + API 文档
- **Code Review**：合并前需要 review
