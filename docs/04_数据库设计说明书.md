# SmartPlant EMS 数据库设计说明书

## 1. 概述

### 1.1 数据库选型

| 项目 | 选择 | 理由 |
|------|------|------|
| **RDBMS** | MySQL 8.0 | 成熟稳定、社区活跃、零成本 |
| **字符集** | utf8mb4 | 支持完整 Unicode（emoji 等） |
| **时序扩展** | 当前用 MySQL 分表 | MVP 阶段够用，后续可升级 TimescaleDB |
| **ORM** | Beego ORM | 框架内置，学习成本低 |

### 1.2 命名规范

| 规范 | 示例 |
|------|------|
| 表名单数 | `device`（不是 `devices`） |
| 字段全小写下划线 | `device_code`（不是 `deviceCode`） |
| 主键统一命名 | `id`（BIGINT AUTO_INCREMENT） |
| 索引命名 | `idx_{table}_{column}` |
| 时间字段 | `created_at`、`updated_at` |

## 3. 表结构详细设计

### 3.1 devices（设备表）

```sql
CREATE TABLE devices (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '设备ID',
    device_code VARCHAR(64) NOT NULL UNIQUE COMMENT '设备编号，如 AC-001',
    device_name VARCHAR(128) NOT NULL COMMENT '设备名称',
    device_type VARCHAR(32) NOT NULL COMMENT '设备类型：air_compressor / injection_machine / laser',
    location VARCHAR(128) COMMENT '安装位置/车间',
    status TINYINT DEFAULT 1 COMMENT '状态：0停用 1运行 2维护 3故障',
    power_rating DECIMAL(10,2) COMMENT '额定功率(kW)',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_device_code (device_code),
    INDEX idx_device_type (device_type),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='设备表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO | 主键 |
| device_code | VARCHAR(64) | NOT NULL, UNIQUE | 设备编号 |
| device_name | VARCHAR(128) | NOT NULL | 设备名称 |
| device_type | VARCHAR(32) | NOT NULL | 设备类型 |
| location | VARCHAR(128) | 可空 | 安装位置 |
| status | TINYINT | DEFAULT 1 | 状态 |
| power_rating | DECIMAL(10,2) | 可空 | 额定功率(kW) |
| created_at | TIMESTAMP | AUTO | 创建时间 |
| updated_at | TIMESTAMP | AUTO | 更新时间 |

**说明**：
- `device_code` 唯一，用于设备身份标识
- `device_type` 在应用层做枚举校验
- 索引 `idx_device_type` 用于按类型筛选
- `status` 使用 TINYINT 节省空间

### 3.2 energy_records（能耗记录表）

```sql
CREATE TABLE energy_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '记录ID',
    device_id BIGINT NOT NULL COMMENT '设备ID',
    record_time TIMESTAMP NOT NULL COMMENT '记录时间',
    power_kw DECIMAL(10,3) COMMENT '瞬时功率(kW)',
    energy_kwh DECIMAL(12,4) COMMENT '累计能耗(kWh)',
    voltage_v DECIMAL(8,2) COMMENT '电压(V)',
    current_a DECIMAL(8,3) COMMENT '电流(A)',
    power_factor DECIMAL(5,3) COMMENT '功率因数',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_device_time (device_id, record_time),
    INDEX idx_record_time (record_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='能耗记录表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO | 主键 |
| device_id | BIGINT | NOT NULL, FK | 设备ID |
| record_time | TIMESTAMP | NOT NULL | 记录时间 |
| power_kw | DECIMAL(10,3) | 可空 | 瞬时功率(kW) |
| energy_kwh | DECIMAL(12,4) | 可空 | 累计能耗(kWh) |
| voltage_v | DECIMAL(8,2) | 可空 | 电压(V) |
| current_a | DECIMAL(8,3) | 可空 | 电流(A) |
| power_factor | DECIMAL(5,3) | 可空 | 功率因数 |

**说明**：
- 高频写入表，建议按月分表（历史数据归档）
- `energy_kwh` 使用 DECIMAL(12,4) 保证精度
- 联合索引 `idx_device_time` 加速按设备和时间范围查询

### 3.3 cost_records（能源成本表）

```sql
CREATE TABLE cost_records (

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_cost_date (cost_date),
    INDEX idx_work_order (work_order)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='能源成本表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO | 主键 |
| cost_date | DATE | NOT NULL | 成本日期 |
| shift | VARCHAR(16) | 可空 | 班次 |
| work_order | VARCHAR(64) | 可空 | 工单号 |
| production_line | VARCHAR(64) | 可空 | 产线 |
| total_energy_kwh | DECIMAL(12,4) | 可空 | 总能耗 |
| unit_price | DECIMAL(8,4) | 可空 | 电价 |
| total_cost | DECIMAL(12,2) | 可空 | 总成本 |

**说明**：
- 按工单/产线统计能源成本
- 支持峰谷平电价分别计算
- 统计粒度：班次级别

### 3.4 alerts（告警表）

```sql
CREATE TABLE alerts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '告警ID',
    device_id BIGINT NOT NULL COMMENT '设备ID',
    alert_type VARCHAR(32) NOT NULL COMMENT '告警类型',
    alert_level TINYINT NOT NULL COMMENT '级别：1普通 2重要 3紧急',
    alert_msg TEXT COMMENT '告警信息',
    threshold_value DECIMAL(10,3) COMMENT '触发阈值',
    current_value DECIMAL(10,3) COMMENT '当前值',
    is_resolved TINYINT DEFAULT 0 COMMENT '是否解决',
    resolved_at TIMESTAMP NULL COMMENT '解决时间',
    resolved_by VARCHAR(64) NULL COMMENT '处理人',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_device_alerts (device_id, is_resolved),
    INDEX idx_alert_level (alert_level),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='告警表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO | 主键 |
| device_id | BIGINT | NOT NULL, FK | 设备ID |
| alert_type | VARCHAR(32) | NOT NULL | 告警类型 |
| alert_level | TINYINT | NOT NULL | 级别 |
| alert_msg | TEXT | 可空 | 告警信息 |
| threshold_value | DECIMAL(10,3) | 可空 | 触发阈值 |
| current_value | DECIMAL(10,3) | 可空 | 当前值 |
| is_resolved | TINYINT | DEFAULT 0 | 是否解决 |
| resolved_at | TIMESTAMP | 可空 | 解决时间 |
| resolved_by | VARCHAR(64) | 可空 | 处理人 |

**告警类型枚举**：
| 值 | 类型 | 说明 |
|------|------|------|
| `over_power` | 功率超限 | 瞬时功率超过额定 |
| `under_power_factor` | 功率因数过低 | 低于 0.85 |
| `voltage_anomaly` | 电压异常 | 超出国标范围 |
| `current_overload` | 电流过载 | 超过额定电流 110% |
| `offline` | 设备离线 | 数据采集中断 |

### 3.5 users（用户表）

```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(64) NOT NULL UNIQUE COMMENT '用户名',
    password_hash VARCHAR(256) NOT NULL COMMENT '密码(bcrypt)',
    real_name VARCHAR(64) COMMENT '真实姓名',
    role VARCHAR(16) NOT NULL DEFAULT 'operator' COMMENT '角色：admin/admin/operator/technician',
    phone VARCHAR(32) COMMENT '手机号',
    email VARCHAR(128) COMMENT '邮箱',
    status TINYINT DEFAULT 1 COMMENT '状态：0禁用 1启用',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_role (role)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO | 主键 |
| username | VARCHAR(64) | NOT NULL, UNIQUE | 用户名 |
| password_hash | VARCHAR(256) | NOT NULL | 密码（bcrypt） |
| real_name | VARCHAR(64) | 可空 | 真实姓名 |
| role | VARCHAR(16) | NOT NULL, DEFAULT 'operator' | 角色 |
| phone | VARCHAR(32) | 可空 | 手机号 |
| email | VARCHAR(128) | 可空 | 邮箱 |
| status | TINYINT | DEFAULT 1 | 状态 |

**角色说明**：
| 角色 | 权限 |
|------|------|
| admin | 全部权限 |
| operator | 数据查看、报表导出 |
| technician | 设备管理、告警处理 |

## 4. 索引设计

### 4.1 索引汇总

| 表名 | 索引名 | 字段 | 类型 | 说明 |
|------|--------|------|------|------|
| devices | PRIMARY | id | BTREE | 主键 |
| devices | idx_device_code | device_code | BTREE | 唯一查询 |
| devices | idx_device_type | device_type | BTREE | 类型筛选 |
| devices | idx_status | status | BTREE | 状态筛选 |
| energy_records | PRIMARY | id | BTREE | 主键 |
| energy_records | idx_device_time | device_id, record_time | BTREE | 组合查询 |
| energy_records | idx_record_time | record_time | BTREE | 时间范围查询 |
| alerts | PRIMARY | id | BTREE | 主键 |
| alerts | idx_device_alerts | device_id, is_resolved | BTREE | 未解决告警查询 |
| alerts | idx_alert_level | alert_level | BTREE | 级别筛选 |
| alerts | idx_created | created_at | BTREE | 时间筛选 |
| cost_records | PRIMARY | id | BTREE | 主键 |
| cost_records | idx_cost_date | cost_date | BTREE | 日期筛选 |
| cost_records | idx_work_order | work_order | BTREE | 工单查询 |
| users | PRIMARY | id | BTREE | 主键 |
| users | idx_username | username | BTREE | 登录查询 |
| users | idx_role | role | BTREE | 角色筛选 |
| reports | PRIMARY | id | BTREE | 主键 |
| reports | idx_report_type | report_type | BTREE | 类型筛选 |
| reports | idx_period | period_start, period_end | BTREE | 时间范围 |

### 4.2 查询优化建议

**高频查询场景及优化**：

1. **查询某设备最新能耗**：
   ```sql
   SELECT * FROM energy_records
   WHERE device_id = 1
   ORDER BY record_time DESC
   LIMIT 1;
   -- 索引: idx_device_time (device_id, record_time) ✅
   ```

2. **查询某时间范围的设备能耗**：
   ```sql
   SELECT * FROM energy_records
   WHERE device_id = 1
     AND record_time BETWEEN '2026-04-01' AND '2026-04-20'
   ORDER BY record_time;
   -- 索引: idx_device_time (device_id, record_time) ✅
   ```

3. **查询未解决的设备告警**：
   ```sql
   SELECT * FROM alerts
   WHERE device_id = 1 AND is_resolved = 0
   ORDER BY created_at DESC;
   -- 索引: idx_device_alerts (device_id, is_resolved) ✅
   ```

## 5. 数据分区（未来扩展）

当前 MVP 阶段暂不分区，当数据量超过 500 万条后建议按月分区：

```sql
-- 分区策略：按 record_time 按月 Range 分区
ALTER TABLE energy_records
PARTITION BY RANGE (UNIX_TIMESTAMP(record_time)) (
    PARTITION p202604 VALUES LESS THAN (UNIX_TIMESTAMP('2026-05-01')),
    PARTITION p202605 VALUES LESS THAN (UNIX_TIMESTAMP('2026-06-01')),
    PARTITION p202606 VALUES LESS THAN (UNIX_TIMESTAMP('2026-07-01')),
    PARTITION future VALUES LESS THAN MAXVALUE
);
```
