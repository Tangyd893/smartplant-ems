# SmartPlant EMS 架构设计说明书

## 1. 概述

### 1.1 文档目的
本文档用于描述 SmartPlant EMS 的系统架构设计，包括技术架构、部署架构、微服务设计、模块划分等内容，为后续开发提供技术指导。

### 1.2 架构原则

| 原则 | 说明 |
|------|------|
| **前后端分离** | 前端 SPA，后端纯 API，无服务端渲染 |
| **微服务拆分** | 按业务领域拆分为独立服务，独立部署 |
| **轻量级** | 不引入额外的中间件，MySQL + Nginx 足够 |
| **可扩展** | 预留 TimescaleDB / Redis / EMQX 升级路径 |
| **容器化** | 所有服务通过 Docker 部署 |

## 3. 微服务设计

### 3.1 服务概览

| 服务 | 端口 | 功能 | 数据库 |
|------|------|------|--------|
| device-service | 8081 | 设备管理 | smartplant_ems.devices |
| energy-service | 8082 | 能耗分析 | smartplant_ems.energy_records |
| report-service | 8083 | 报表管理 | smartplant_ems.reports |

### 3.2 服务间调用关系

```
前端 (3000)
    │
    └─ HTTP → Nginx (:8080)
                  │
                  ├─ /api/device/* → device-service (:8081)
                  ├─ /api/energy/* → energy-service (:8082)
                  └─ /api/report/* → report-service (:8083)

服务间调用（可选，按需）:
device-service → energy-service (获取设备能耗统计)
energy-service → report-service (推送统计数据用于报表生成)
```

### 3.3 Namespace 路由设计

```go
// device-service 路由
Namespace("/api/device",
    Router("/list", DeviceController{}, "Get:List"),
    Router("/:id", DeviceController{}, "Get:Get,Put:Update,Delete:Delete"),
    Router("", DeviceController{}, "Post:Create"),
    Router("/:id/realtime", DeviceController{}, "Get:GetRealTimeData"),
)

// energy-service 路由
Namespace("/api/energy",
    Router("/list", EnergyController{}, "Get:List"),
    Router("/:id", EnergyController{}, "Get:Get,Put:Update,Delete:Delete"),
    Router("/stats", EnergyController{}, "Get:GetStats"),
)

// report-service 路由
Namespace("/api/report",
    Router("/list", ReportController{}, "Get:List"),
    Router("/:id", ReportController{}, "Get:Get"),
    Router("/:id/download", ReportController{}, "Get:Download"),
    Router("", ReportController{}, "Post:Create"),
    Router("/:id", ReportController{}, "Delete:Delete"),
)
```

## 4. 部署架构

### 4.1 Docker Compose 部署

```yaml
# docker/docker-compose.yml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: sp_mysql
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: smartplant_ems
      MYSQL_USER: spuser
      MYSQL_PASSWORD: sp123456
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping"]
      interval: 10s
      retries: 5

  nginx:
    image: nginx:alpine
    container_name: sp_nginx
    ports:
      - "80:80"
      - "8080:8080"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      - frontend_dist:/usr/share/nginx/html
    depends_on:
      - mysql

volumes:
  mysql_data:
  frontend_dist:
```

### 4.2 后端服务 Dockerfile

```dockerfile
# backend/device-service/Dockerfile
FROM alpine:3.19
COPY device-service-bin /app/device-service
WORKDIR /app
EXPOSE 8081
CMD ["/app/device-service"]
```

> 注：后端服务在 MVP 阶段直接通过宿主机二进制运行，不纳入 docker-compose 管理，后续迭代中纳入。

## 6. 数据流设计

### 6.1 能耗数据采集流程

```
设备 → HTTP POST /api/energy → energy-service → MySQL (energy_records)
```

数据由设备通过 HTTP 轮询推送，频率建议 ≤ 1次/分钟。

### 6.2 告警触发流程

```
设备数据 → 超过阈值 → 创建 Alert 记录 → 前端轮询 / WebSocket → 用户收到通知
```

### 6.3 报表生成流程

```
用户请求生成报表 → report-service 创建任务 → 异步查询 energy_service 统计 → 生成 Excel → 保存 FilePath → 用户下载
```

## 7. 安全设计

### 7.1 认证机制（预留）

MVP 阶段暂不实现，生产阶段应引入 JWT：

```
用户登录 → 验证用户名密码 → 签发 JWT → 前端存储 → 请求时 Header 携带
```

### 7.2 Nginx 安全头

```nginx
add_header X-Frame-Options "SAMEORIGIN";
add_header X-Content-Type-Options "nosniff";
add_header X-XSS-Protection "1; mode=block";
```

### 7.3 CORS 配置

```nginx
# 只允许本地域名访问
add_header Access-Control-Allow-Origin "http://localhost:3000";
```

## 8. 性能优化策略

### 8.1 前端优化

| 策略 | 说明 |
|------|------|
| 代码分割 | Vite 懒加载页面组件 |
| Gzip 压缩 | Nginx 开启 gzip |
| 静态资源缓存 | 静态资源设置 long cache |

### 8.2 后端优化

| 策略 | 说明 |
|------|------|
| 数据库索引 | 对高频查询字段建立索引 |
| 分页查询 | 默认 page size=20，避免全表扫描 |
| 连接池 | MySQL 连接池复用 |

### 8.3 Nginx 优化

```nginx
gzip on;
gzip_types text/plain application/json application/javascript text/css;
gzip_min_length 1024;
```
